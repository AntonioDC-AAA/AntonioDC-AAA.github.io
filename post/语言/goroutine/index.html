协程Goroutine

轻量级“线程”

非抢占式多任务处理，由协程主动交出控制权

编译器/解释器/虚拟机层面的多任务

多个协程可能在一个或多个线程上运行

抢占式多任务会造成卡死的情况，这时需要：

1.做IO操作会交出控制权

2.runtime.Gosched()手动交出控制权

goroutine可能的切换点

  • I/O ,select
  • Channel
  • 等待锁
  • 函数调用（有时候）
  • runtime.Gosched()

只是参考，不能保证切换，不能保证在其他地方不切换

go run -race [name.go] //查看数据冲突的方法
